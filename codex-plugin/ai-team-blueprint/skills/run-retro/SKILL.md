# Run Retro

1. State the observed event and its impact using evidence.
2. Build a blameless timeline and identify the system cause, not the nearest
   person or superficial symptom.
3. Name the missing or ineffective signal that allowed the issue through.
4. Select one durable landing place:
   - `AGENTS.md` for always-loaded behavioral guidance;
   - a skill for a reusable workflow;
   - a hook, test, linter, permission, or CI job for enforceable rules;
   - project documentation for canonical facts.
5. Prefer a mechanical gate when the rule must never slip. Verify the failure
   case actually triggers the gate.
6. Keep the change narrow. Do not create a new agent when an existing role plus
   a checklist or skill can own the learning.
7. Record the owner and review date for any temporary exception.

Return the learning, the chosen landing spot, the applied or proposed change,
and its verification receipt.
